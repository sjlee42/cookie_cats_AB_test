## 1. 프로젝트 배경

### 1-1. Cookie Cats 게임 소개

Cookie Cats는 "connect-three" 스타일의 F2P 모바일 퍼즐 게임이다. 게임 중 특정 레벨에 도달하면 **"Gate"** 라는 강제 대기 메커니즘이 작동하여, 유저는 일정 시간을 기다리거나 인앱 구매를 해야 다음 레벨로 진행할 수 있다. Gate는 두 가지 기능을 한다:

- **수익화:** 대기를 건너뛰기 위한 인앱 구매 유도
- **세션 조절:** 지나친 플레이로 인한 피로감 방지 → 장기 잔존율 개선

### 1-2. 실험 목적

현재 Gate는 레벨 30에 위치한다. Gate를 레벨 40으로 이동하면 유저가 더 많은 콘텐츠를 먼저 경험하게 되어 잔존율이 높아질 수 있다는 가설이 있다. 이 A/B 테스트는 해당 가설을 데이터로 검증한다.

### 1-3. 의사결정 프레임워크

| 구분 | 기준 |
|------|------|
| **Primary KPI** | D7 잔존율 차이 (gate_30 vs gate_40), p < 0.05 |
| **Secondary KPI** | D1 잔존율 차이, 게임 라운드 수 중앙값 차이 |
| **방향** | gate_30과 gate_40 중 어느 쪽이 더 높든 그 방향으로 의사결정 |
| **통계적 실패** | p > 0.05이거나 Bootstrap 신뢰구간이 0을 포함하면 → 차이가 우연일 가능성 |
| **실용적 실패** | 통계적으로 유의해도 효과 크기가 작으면 → 변경 비용 대비 실익 없음 |

### 1-4. 이해관계자

| 이해관계자 | 관심 포인트 |
|-----------|------------|
| **게임 기획팀** | 게이트 위치 결정 → 난이도 곡선 설계의 기준 |
| **마케팅/그로스 팀** | D1/D7 잔존율은 UA(사용자 획득) 캠페인 ROI 계산의 핵심 지표 |
| **경영진** | 설계 변경 없이 잔존율을 개선할 수 있는지 데이터로 판단 |
| **유저** | 게임이 더 오래 운영되고, 플레이어 풀이 유지됨 |

## 2. 데이터 및 전처리

### 2-1. 원본 데이터

| 항목 | 값 |
|------|-----|
| 파일 | `data/cookie_cats.csv` |
| 총 행 수 | 90,189명 |
| 관찰 기간 | 설치 후 14일 |

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `userid` | int | 고유 유저 ID |
| `version` | str | A/B 그룹 — `gate_30` (대조군) / `gate_40` (실험군) |
| `sum_gamerounds` | int | 설치 후 14일간 플레이한 총 라운드 수 |
| `retention_1` | bool | 설치 다음날 접속 여부 (D1 잔존율) |
| `retention_7` | bool | 설치 7일 후 접속 여부 (D7 잔존율) |

### 2-2. 전처리 (ITT 기준)

```python
OUTLIER_UPPER = 30_000
df = df_raw[df_raw['sum_gamerounds'] <= OUTLIER_UPPER].copy()  # 극단 이상치 제거
df = df[df['sum_gamerounds'] > 0].copy()                       # 0라운드 유저 제거
```

| 단계 | 제거 대상 | 제거 수 | 잔여 |
|------|----------|:-------:|:----:|
| 원본 | — | — | 90,189명 |
| 0라운드 제거 | 설치 후 미실행 유저 | 3,994명 | 86,195명 |
| 극단치 제거 | sum_gamerounds > 30,000 | 1명 | **86,194명** |

**제거 근거:**

- **0라운드 유저:** 앱을 설치만 하고 단 한 번도 실행하지 않은 유저. Gate를 경험할 수 없으므로 A/B 테스트 대상이 아님.
- **30,000라운드 초과:** 14일간 30,000라운드는 정상적 플레이로 보기 어려운 극단값 (1라운드 180초 가정 시 하루 25시간 이상). 분석 왜곡 방지 목적.

**중요:** 본 분석의 기본 프레임워크는 **ITT(Intention-To-Treat)** 원칙을 따른다. 게이트 도달 여부와 무관하게, 무작위 배정된 전체 유저를 분석 대상으로 포함한다. 게이트 통과 유저만 분석하는 Per-Protocol 접근은 8장에서 별도로 논의한다.

## 4. 가설 검정 — 기본 분석

> 전처리 기준: 0라운드 유저 제거 + sum_gamerounds ≤ 30,000 → **86,194명**

### 4-1. 기술 통계 (전처리 후)

| 지표 | gate_30 (42,762명) | gate_40 (43,432명) | 차이 (30 − 40) |
|------|:---:|:---:|:---:|
| D1 잔존율 | 46.75% | 46.22% | **+0.54%p** |
| D7 잔존율 | 19.84% | 19.03% | **+0.81%p** |

![잔존율 비교](figures/03_retention_comparison.png)

### 4-2. Primary KPI: D7 잔존율

#### Chi-square 독립성 검정

| 통계량 | 값 |
|--------|-----|
| χ² | 8.9849 |
| 자유도 (df) | 1 |
| p-value | **0.0027** |
| 판정 | ✅ **유의 (p < 0.05)** |

#### Bootstrap 신뢰구간 (10,000회 반복)

![Bootstrap D7](figures/04_bootstrap_d7.png)

| 항목 | 값 |
|------|-----|
| 평균 차이 (gate_30 − gate_40) | +0.808%p |
| 95% CI | **[+0.269%p, +1.351%p]** |
| 0 포함 여부 | ❌ **포함하지 않음 → 차이 유의** |

#### 순열 검정 (Permutation Test, 10,000회 반복)

| 항목 | 값 |
|------|-----|
| 관찰된 차이 (gate_30 − gate_40) | +0.8103%p |
| 순열 귀무분포 평균 | ≈ 0 (기대값) |
| p-value | **0.0023** |
| 판정 | ✅ **유의 (p < 0.05)** |

순열 검정은 그룹 레이블을 무작위로 섞어 귀무분포(null distribution)를 직접 구성하는 비모수 검정이다. Bootstrap이 원래 그룹 내 재표본 추출인 것과 달리, 순열 검정은 레이블 자체를 섞어 "그룹 배정이 무의미할 때" 어떤 차이가 생기는지 시뮬레이션한다. 분포 가정이 전혀 없는 정확 검정(exact test)에 가장 가까운 방법이다.

**4-2 검정 결과 종합**

| 검정 방법 | 통계량 | p-value | H0 기각 | 95% CI |
|----------|--------|:-------:|:-------:|--------|
| 카이제곱 | χ² = 8.9849 | 0.0027 | ✅ | — |
| Two-proportion Z-test | Z = 3.0061 | 0.0026 | ✅ | [+0.2820%p, +1.3387%p] |
| Bootstrap (10,000회) | diff = +0.8103%p | 0.0033 | ✅ | [+0.2686%p, +1.3507%p] |
| 순열 검정 (10,000회) | diff = +0.8103%p | 0.0023 | ✅ | — |

**해석:** D7 잔존율은 gate_30이 통계적으로 유의하게 높다. 네 가지 검정 방법 모두 일관되게 H0을 기각한다.

### 4-4. Secondary KPI: 게임 라운드 수 (Mann-Whitney U)

게임 라운드 분포는 극심한 우측 편중이므로 t-test 대신 비모수 검정인 Mann-Whitney U를 사용했다.

#### Mann-Whitney U 검정

| 통계량 | 값 |
|--------|-----|
| U | 934,332,123 |
| p-value | **0.1177** |
| 판정 | ❌ **유의하지 않음** |

| 그룹 | 중앙값 | 평균 | Q1 | Q3 |
|------|:------:|:----:|:--:|:--:|
| gate_30 | 18.0 | 53.7 | 5.0 | 51.0 |
| gate_40 | 18.0 | 53.7 | 5.0 | 51.0 |

**해석:** 두 그룹의 게임 라운드 분포는 통계적으로 유의한 차이가 없다. 게이트 위치 변경이 플레이량 자체에는 영향을 미치지 않았다.

1. **D7 잔존율은 gate_30이 유의하게 높다.** p = 0.0027, Bootstrap 95% CI가 0을 포함하지 않음.
2. **gate_40으로 변경할 이유가 없다.** 세 가지 KPI 모두에서 gate_40의 장점 없음.
3. **효과 크기는 매우 작다.** Cohen's h = 0.02. 변경 비용 대비 개선 가능성 0.
4. **민감도 분석으로 강건성 확인.** 4가지 이상치 기준 모두 동일 결론.

## 7. 부록: 분석 방법론

### 7-1. 통계 검정 요약

| 검정 | 대상 | 목적 | 선택 이유 |
|------|------|------|----------|
| Chi-square 독립성 검정 | D7, D1 잔존율 | 두 비율의 차이가 우연인지 검증 | 범주형 변수 간 독립성 검정의 표준 방법 |
| Two-proportion Z-test | D7 잔존율 | 비율 차이에 대한 정규근사 검정 + 신뢰구간 산출 | 신뢰구간을 직접 계산할 수 있어 방향·크기 해석이 용이 |
| Bootstrap (10,000회) | D7, D1 잔존율 | 비모수적 신뢰구간 추정 | 분포 가정 없이 강건한 추정 가능 |
| 순열 검정 (10,000회) | D7 잔존율 | 귀무분포를 직접 시뮬레이션하여 p-value 산출 | 분포 가정 없는 정확 검정, Bootstrap과 상호 검증 |
| Mann-Whitney U | 게임 라운드 수 | 비정규 분포의 위치 차이 검정 | 극심한 우측 편중 분포에 t-test 부적합 |
| Cohen's h | D7, D1 잔존율 | 실질적 효과 크기 평가 | 비율 차이의 표준 효과 크기 측정 |

### 7-2. Cohen's h 계산식

```
h = 2 × arcsin(√p₁) − 2 × arcsin(√p₂)
```

| 기준 | 효과 크기 |
|------|----------|
| |h| < 0.2 | Small (작음) |
| 0.2 ≤ |h| < 0.5 | Medium (중간) |
| |h| ≥ 0.8 | Large (큼) |

### 7-3. Bootstrap vs 순열 검정 차이

| 구분 | Bootstrap | 순열 검정 |
|------|-----------|----------|
| **레이블 처리** | 원래 그룹 유지, 그룹 내 재표본 | 레이블을 섞어 그룹 재할당 |
| **목적** | 신뢰구간 추정 (표본분포 추정) | 귀무분포 직접 구성 (p-value) |
| **귀무가설** | 명시적 귀무 분포 없음 | H₀("그룹 레이블 무관") 직접 시뮬레이션 |
| **p-value 계산** | 귀무 중심 이동 후 극단치 비율 | 귀무분포 상 관찰값보다 극단인 비율 |

### 7-4. 순열 검정 절차

```python
np.random.seed(42)
n_iterations = 10_000

pool = df['retention_7'].values  # 전체 풀 (레이블 무관)

for each iteration:
    shuffled = np.random.permutation(pool)
    perm_diff = shuffled[:n30].mean() - shuffled[n30:n30+n40].mean()
    store perm_diff

p_value = mean(|perm_diffs| >= |observed_diff|)
```

### 7-5. Bootstrap 절차

```python
np.random.seed(42)
n_iterations = 10_000

for each iteration:
    sample_g30 = resample gate_30 group with replacement (n = |gate_30|)
    sample_g40 = resample gate_40 group with replacement (n = |gate_40|)
    diff = mean(sample_g30.retention_7) - mean(sample_g40.retention_7)
    store diff

CI_95 = percentile(diffs, [2.5, 97.5])
```

## 8. 인과 추론 논의 — ITT vs Per-Protocol

이 장은 본 분석에서 가장 중요한 방법론적 논의를 다룬다. **전처리 방식 하나로 결론이 완전히 뒤집히는 현상**을 분석하고, 왜 ITT 기준 분석만이 인과적 해석이 가능한지를 설명한다.

### 8-1. 문제 제기: "게이트를 경험한 유저만 분석해야 하지 않을까?"

직관적으로 다음과 같은 전처리가 합리적으로 보일 수 있다:

```python
# Per-Protocol 전처리: 게이트를 "경험한" 유저만 남기기
df_pp = df[
    ((df['version']=='gate_30') & (df['sum_gamerounds'] >= 30)) |
    ((df['version']=='gate_40') & (df['sum_gamerounds'] >= 40))
]
```

gate_30 유저 중 30라운드 미만, gate_40 유저 중 40라운드 미만 플레이어를 제외한다. 게이트를 경험조차 못 한 유저가 포함되면 "게이트 위치가 retention에 미치는 영향"을 제대로 측정할 수 없다는 논리이다.

**그러나 이 전처리는 생존 편향(Survivorship Bias)을 유발하여 결론을 정반대로 뒤집는다.**

### 8-2. 실증 비교: ITT vs Per-Protocol 결과

| 분석 유형 | 대상 | gate_30 N | gate_40 N |
|-----------|------|:---------:|:---------:|
| **ITT** (전체 유저) | 배정된 모든 유저 | 42,762 | 43,432 |
| **Per-Protocol** (통과 유저만) | 게이트 도달 유저만 | 16,655 | 13,827 |

#### D1 잔존율

| 분석 | gate_30 | gate_40 | 차이 | p-value | **방향** |
|------|:------:|:------:|:----:|:------:|:------:|
| ITT | 46.75% | 46.22% | +0.54%p | 0.1155 | gate_30 우위 |
| Per-Protocol | 80.10% | **83.02%** | **−2.92%p** | **< 0.001** | **gate_40 우위 (뒤집힘!)** |

#### D7 잔존율

| 분석 | gate_30 | gate_40 | 차이 | p-value | **방향** |
|------|:------:|:------:|:----:|:------:|:------:|
| ITT | 19.84% | 19.03% | +0.81%p | 0.0027 | gate_30 우위 |
| Per-Protocol | 43.87% | **48.51%** | **−4.63%p** | **< 0.001** | **gate_40 우위 (뒤집힘!)** |

**전처리 하나로 결론이 완전히 역전되었다.** ITT에서는 gate_30이 우위였으나, Per-Protocol에서는 gate_40이 압도적으로 우위이며, 그 차이(−4.63%p)는 매우 크고 통계적으로도 강하게 유의하다.

### 8-3. 왜 결론이 뒤집히는가: 생존 편향

#### 무작위 배정이 보장하는 것

A/B 테스트에서 무작위 배정은 다음을 보장한다:

```
gate_30 그룹 전체 ≈ gate_40 그룹 전체
(나이, 게임 실력, 성향, 모든 것이 평균적으로 동일)
```

이 상태에서 두 그룹의 retention 차이 = 게이트 위치의 인과적 효과라고 말할 수 있다.

#### 자르고 나면 무작위성이 깨진다

게이트 통과 유저만 남기면:

```
gate_30 통과 유저 = 전체 중 30라운드 넘긴 사람들 (38.9% 통과)
gate_40 통과 유저 = 전체 중 40라운드 넘긴 사람들 (31.8% 통과)
```

40라운드를 넘기는 것이 30라운드를 넘기는 것보다 더 어렵다. 따라서:

```
gate_30 통과 유저: 비교적 평범한 유저들도 포함
gate_40 통과 유저: 상대적으로 게임을 더 열심히 하는 유저들만 남음
```

무작위 배정 전에는 두 그룹이 동일했는데, **자르고 나니까 게임 열심히 하는 정도(engagement level)가 달라졌다.** 이것이 **사후 처리 변수(post-treatment variable)에 의한 조건화**가 만드는 생존 편향이다.

#### 생존 편향의 크기

| 지표 | gate_30 | gate_40 |
|------|:------:|:------:|
| 게이트 도달 라운드 | 30 | 40 |
| 통과율 | **38.9%** | **31.8%** |
| 통과 유저 수 | 16,655명 | 13,827명 |

gate_40의 통과율이 7.1%p 낮다. gate_40에서 탈락한 유저들은 "게이트를 경험하지 못한" 것이 아니라, **원래 게임을 일찍 포기하는 유저**들이다. 이들을 제거하면 gate_40 그룹에는 자연스럽게 충성도가 높은 유저만 남게 된다.

### 8-4. 올바른 해석 프레임워크

| 분석 유형 | 해석 수준 | 말할 수 있는 것 | 말할 수 없는 것 |
|-----------|----------|----------------|----------------|
| **ITT (분석 1)** | **인과적** | "gate_30 배정이 retention을 높인다" | — |
| **Per-Protocol (분석 2)** | **상관적 (탐색적)** | "gate_40 통과 유저는 retention이 높은 경향이 있다" | "gate_40이 retention에 더 좋다" |

Per-Protocol에서 gate_40이 retention이 높다 해도, 이것이 **게이트를 40으로 늦춘 덕분인지, 아니면 원래 열심히 하는 유저들이 많이 남아있어서인지 구분할 수 없다.**

### 8-5. 결론

```
전처리 방식에 따라 결론이 뒤집혔다.
이는 sum_gamerounds가 사후 처리 변수(post-treatment variable)이기 때문에 발생한 편향이다.
인과적 해석은 ITT 기준 분석을 따라야 한다.
```

**ITT 기준 최종 결론:** gate_30이 retention에 유리하다.

**Per-Protocol은 보조 분석으로만 활용한다:** "gate_40을 통과한 유저 집단의 특성이 retention과 어떤 관계에 있는지"에 대한 탐색적 정보를 제공하지만, 이 결과로 게이트 위치를 변경하는 의사결정을 내려서는 안 된다.
